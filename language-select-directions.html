1 Compact — Language flyout from profile click

Zero new surfaces. Profile click opens a small popup directly above the footer row. Logout stays in both places — popup and the existing icon button.

Layout

Profile row avatar + name become a unified clickable zone (separate from the logout icon button, which retains its existing tap target). A two-row popup (218px wide) animates in just above: row 1 = Language with current locale badge + chevron, row 2 (after divider) = Logout in pink. Clicking Language expands a flyout sub-list to the side with 6 locale options + flag + checkmark on active.

Components Used

  • Popup (card shell — white, radius-md, shadow-popup)
  • Sidebar nav item pattern (for popup rows)
  • Filter Tags (locale badge pill: gray-100 bg)
  • Logout row reuses existing icon + Popup confirm

Tokens

--white--blue #1670EE --pink #E91E8C--g100 (badge bg) --sh-popup--r-md 12px font 13px rows / 10px badge

Interaction

  • Click avatar+name → language list opens immediately
  • Select locale → badge + subtitle update, list closes
  • Click outside → list closes
  • Logout icon in footer row works independently
Rationale: The logout icon already exists and works — this design respects that affordance rather than removing it. Language selection is a one-time configuration, so a two-level flyout (click profile → click Language → pick) is acceptable friction. The badge ("EN") gives persistent ambient feedback about current locale without adding any new chrome. The profile row subtitle "🌐 EN · click to change" acts as a discoverable CTA without disrupting the existing visual hierarchy.
Best for: Power users · bilingual analysts · returning users who configure once

2 Balanced — In-sidebar drawer from profile

Medium footprint. Profile click lifts a drawer panel inside the sidebar — no popup leaving the column. Language chips are immediately visible. Logout at the bottom of the drawer.

Layout

The profile row (avatar + name) is clickable. On tap, a bottom drawer slides up from within the sidebar footer zone, covering the nav history area. It shows: a mini profile header (avatar, name, close ✕); a grey section label "🌐 Interface Language"; a 2-column chip grid with flag + short name for 6 languages; and a full-width Logout button in pink at the base. The existing logout icon button remains visible when the drawer is closed.

Components Used

  • Popup (card shell — same shadow & radius tokens)
  • Filter Tags pattern (language chip: border, hover, selected states)
  • Section Header token (gray-50 bg, uppercase label)
  • Profile mini — Avatar + name pattern from Sidebar Footer
  • Logout button — uses pink/danger token set

Tokens

--blue-lt (chip selected bg) --g200 (chip border) --g50 (section header bg) --pink + pink-lt (logout) 0 -8px 32px shadow (drawer) --r-lg 16px corners

Interaction

  • Subtitle "🇬🇧 English · tap to change" is always visible
  • Click avatar+name → drawer slides up
  • Tap chip → instant selection + ✓, subtitle updates
  • Click ✕ or tap outside → drawer collapses
  • Logout button → existing confirmation Popup
Rationale: Keeping the drawer inside the sidebar column means it never disrupts the main content area and doesn't feel like a modal interruption. The chip grid reuses the Filter Tags pattern already in the component library — no new component needed. The persistent subtitle under the username provides instant ambient context (current language) and a clear affordance to change it. Logout being inside the drawer removes the redundancy of the icon button — in this direction, the icon button is a fallback for users who haven't discovered the drawer.
Best for: International teams · first-time users exploring settings · daily users

3 Spacious — Sidebar nav swaps to settings view

Maximum clarity. Profile click transitions the entire sidebar nav area to a settings screen. Language list is full-height with radio buttons and native names. Logout at the bottom.

Layout

The sidebar nav zone (everything between the app pill and the footer) slides left and is replaced by a settings view using the same column width. The view has: a header bar with a Back arrow + "Settings" title; a profile block (avatar, name, plan); then the full language list — each row shows a flag emoji, English name, native name, and a radio button. Below a divider, a Logout row in pink. A "Save Language" CTA sits above the footer credits card.

Components Used

  • Sidebar nav item (repurposed as language option rows)
  • Section Header (for "Interface Language" label)
  • Profile Icon (large, from Sidebar Footer component)
  • Radio button — blue token on selected state
  • Primary button (Save) — blue, full-width
  • Logout row — pink, same pattern as nav item

Tokens

--blue (button + radio) --blue-lt (row selected bg) --pink + pink-lt (logout) --g100 (divider, borders) --g50 (section header bg) transition: 220ms cubic-bezier

Interaction

  • Footer subtitle "⚙️ Settings & Language →" is persistent cue
  • Click avatar+name → nav slides out, settings slides in
  • Tap a language row → radio fills, row highlights blue
  • Click "Save Language" → slides back, subtitle updates
  • Back arrow → discards pending change, slides back
  • Logout in settings OR logout icon in footer both work
Rationale: The Session Side Panel component pattern in the library already uses this "slide-in replacement" interaction for session detail views — this direction extends that established grammar to settings. Full-height language rows with native names (日本語, 한국어) give non-Latin scripts adequate legibility that chips cannot provide at small sizes. The Back arrow makes the transition feel safe and reversible. The "Save Language" button prevents accidental language switching on mis-tap, which matters more here than in the chip pattern.
Best for: Admins · onboarding users · multi-region teams with non-Latin scripts
